- MYSQL索引失效的各种情形总结
```
1) 没有查询条件，或者查询条件没有建立索引 
2) 查询的数量是大表的大部分，应该是30％以上。
3) 查询条件使用函数在索引列上，或者对索引列进行运算，运算包括(+，-，*，/，! 等)或者使用函数（5.7对索引列进行运算.可以建立函数索引），索引类型会降级为index;
4) 对小表查询
5)隐式转换导致索引失效.这样会导致索引查询类型降级为index ;
6) <>、not导致索引查询类型降级为index
7) like "%_" 百分号在前.
8) 表没分析，数据删除时数据文件和索引文件不回收，可以使用optimize table分析
9) 最左前缀原则.
```
